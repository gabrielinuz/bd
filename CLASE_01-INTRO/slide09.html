09. Antecedentes del SGBD

Sistemas de archivos tradicionales.

Antes de la estandarización de los SGBD, la persistencia se gestionaba mediante sistemas de ficheros individuales exclusivos para cada aplicación.

Principales deficiencias estructurales:

  • Redundancia crítica: Datos repetidos innecesariamente en múltiples archivos.
  • Inconsistencia: Dificultad para sincronizar actualizaciones entre aplicaciones.
  • Aislamiento: Los datos de una aplicación no son accesibles para otra sin procesos de exportación manual.
[ APP_VENTAS ] -> (archivo_clientes.dat)
[ APP_RRHH ] -> (archivo_empleados.dat)
