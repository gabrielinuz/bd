12. Operadores Avanzados

LIKE e IN: Flexibilidad en la búsqueda.

LIKE: Búsqueda de patrones parciales usando comodines (%).

SELECT * FROM Users WHERE nombre LIKE 'Gab%';

IN: Verifica si un valor pertenece a una lista predefinida.

SELECT * FROM Users WHERE pais IN ('AR', 'UY', 'CL');

Rigor técnico: LIKE es costoso en términos de CPU si el comodín está al inicio ('%abc'), ya que impide el uso eficiente de índices.
